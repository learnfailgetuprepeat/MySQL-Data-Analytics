# MySQL Data Analytics Portfolio: Maven Movies Database Audit

## 🏢 Business Scenario Overview
I have assumed day-to-day data operations for **Maven Movies**, a legacy brick-and-mortar DVD rental business. With access to a relational database containing 16 interconnected tables, my objective is to independent analyze operations, investigate customer purchasing habits, manage inventory availability and audit staff performance to streamline business decision-making.

## 📊 Project Roadmap & SQL Application
The analysis tracks the course framework to solve operational problems using structured MySQL queries:

| Analysis Module | Business Objective | Core SQL Competencies |
| :--- | :--- | :--- |
| 📁 **01-Single-Table-Analysis** | Auditing basic inventory counts, film listings, rental rates, and initial customer breakdowns. | `SELECT`, `WHERE`, `GROUP BY`, `HAVING`, Aggregations |
| 📁 **02-Mid-Course-Project** | Consolidating early operational metrics into a comprehensive corporate overview for stakeholders. | Data Summarization, Sub-queries |
| 📁 **03-Multi-Table-Joins** | Mapping relationships across all 16 tables to connect customers to specific films, stores, and staff members. | `INNER JOIN`, `LEFT JOIN`, Multiple Joins, `UNION` |
| 📁 **04-Final-Project** | Delivering an end-to-end data audit and executive brief outlining performance optimization strategies. | Advanced Relational Joins, Complex Aggregations |

## 🛠️ Database Infrastructure
* **Engine:** MySQL
* **Schema Size:** 16 Relational Tables (Tracking Inventory, Rental Transactions, Actor Registries, Customers, and Staff)
